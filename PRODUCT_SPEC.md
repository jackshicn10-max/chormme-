# Product Spec

## 1. 背景

用户在 YouTube 上看英文技术视频学习英语。现有 Language Reactor 可以帮助理解字幕，但它的字幕 UI 不适合沉淀个人生词：

- 字幕文字不稳定可选中。
- 右侧字幕列表由 LR 控制，插件无法可靠接管交互。
- 保存生词时很难同时拿到当前句、上下文、视频时间点。
- 导出和复习不属于用户自己的数据闭环。

本项目的方向是：自己实现一个轻量版 LR 字幕面板，只做学习生词所需的核心功能。

## 2. 产品目标

在 YouTube 播放页中，扩展自动显示一个右侧字幕面板。这个面板包含完整英文字幕，跟随视频播放高亮当前字幕。用户在字幕面板中右键单词即可保存生词。保存记录包含：

- 单词原文
- 归一化单词
- 当前字幕句
- 前后上下文
- 视频标题
- YouTube videoId
- 带时间戳的视频链接
- 保存时间

用户可以打开生词列表，并导出保存记录。

## 3. 核心用户流程

### 3.1 看视频时保存生词

1. 用户打开 YouTube 视频。
2. 扩展自动拉取英文字幕。
3. 页面右侧出现扩展自己的字幕面板。
4. 当前播放位置对应的字幕自动高亮。
5. 用户看到不认识的单词，例如 `representative`。
6. 用户在字幕面板里右键这个单词。
7. 扩展立即保存：
   - word: `representative`
   - sentence: 当前字幕段
   - context: 当前段前后各 2 段
   - videoUrl: `https://www.youtube.com/watch?v=...&t=123s`
8. UI 显示一个简短 toast：`已保存 representative`。

### 3.2 回看上下文

1. 用户打开扩展的生词本。
2. 按视频分组或按时间倒序查看记录。
3. 点击某条记录的视频链接。
4. 回到 YouTube 原时间点。

### 3.3 导出

1. 用户打开生词本。
2. 点击导出。
3. 选择 CSV 或 JSON。
4. 扩展下载文件。

## 4. M1 必须包含

- YouTube 视频页注入右侧字幕面板。
- 拉取当前视频完整英文字幕。
- 字幕列表可滚动。
- 当前字幕跟随播放高亮。
- 字幕中的单词由扩展自己渲染，可以右键。
- 右键单词立即保存生词。
- 保存时带当前句和前后上下文。
- 本地 IndexedDB 持久化。
- 生词本基础页面。
- CSV / JSON 导出。

## 5. M1 不做

- 不做 Language Reactor 兼容。
- 不读取 LR 面板 DOM。
- 不要求 YouTube 原生字幕可选中。
- 不做悬停中文释义。
- 不做点击详细解释。
- 不做 Anki 导出。
- 不做 SRS 复习。
- 不做云同步。
- 不做 Netflix / Bilibili。
- 不做移动端。
- 不做主题/字体/字号自定义。面板样式 M1 固定，后续阶段再开放配置。
- 不做快捷键。M1 只保留鼠标右键保存一条主路径，不加 Alt+S 之类的键盘触发。
- 不做多语言 UI。M1 所有文案中文直写，不引 i18n 框架。

## 6. M2 目标

M2 加入悬停中文简明释义：

- 鼠标悬停单词显示简短中文解释。
- 解释必须结合当前句上下文。
- 解释结果本地缓存。
- 没有 API key 时，M1 功能仍然完整可用。

## 7. M3 目标

M3 加入点击详细解释：

- 点击单词打开详细解释面板。
- 显示词性、上下文含义、常见释义、例句、整句翻译。
- 支持保存解释到生词记录。

## 8. 体验原则

- 保存动作必须快，不能等待翻译。
- 翻译失败不能影响保存。
- 面板必须能关闭和重新打开。
- 字幕面板不应该破坏 YouTube 播放器基本操作。
- 所有学习数据默认只存在本地。
